GET:
Назначение: Запрос данных с сервера.
Примеры использования: Получение страницы, получение данных из API.
Характеристики: Идемпотентный (повторный запрос не изменяет состояние сервера), безопасный (не изменяет данные на сервере).

POST:
Назначение: Отправка данных на сервер для создания ресурса.
Примеры использования: Отправка формы регистрации, создание новой записи в базе данных.
Характеристики: Неидемпотентный (повторный запрос может создать новый ресурс), данные отправляются в теле запроса.

PUT:
Назначение: Отправка данных на сервер для создания или обновления ресурса.
Примеры использования: Обновление профиля пользователя, замена файла.
Характеристики: Идемпотентный (повторный запрос приводит к одному и тому же результату), данные отправляются в теле запроса.

DELETE:
Назначение: Удаление ресурса на сервере.
Примеры использования: Удаление учетной записи пользователя, удаление файла.
Характеристики: Идемпотентный.

PATCH:
Назначение: Частичное обновление ресурса на сервере.
Примеры использования: Обновление части профиля пользователя, изменение отдельных свойств объекта.
Характеристики: Неидемпотентный.

HEAD:
Назначение: Запрос заголовков HTTP без тела ответа.
Примеры использования: Проверка существования ресурса, получение метаинформации.
Характеристики: Идемпотентный, безопасный.

OPTIONS:
Назначение: Запрос доступных методов и возможностей для ресурса на сервере.
Примеры использования: Проверка поддерживаемых методов API, CORS-запросы.
Характеристики: Идемпотентный, безопасный.

CONNECT:
Назначение: Устанавливает туннель для передачи данных, обычно используется для прокси-серверов.
Примеры использования: Установка туннеля через прокси.
Характеристики: Неидемпотентный.

TRACE:
Назначение: Диагностика пути запроса до сервера, возвращает запрос как получил сервер.
Примеры использования: Трассировка пути запроса.
Характеристики: Идемпотентный, но не безопасный (может быть использован для атак).
